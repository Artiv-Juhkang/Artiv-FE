# AppToon — 제품 로드맵 & 수익 모델 설계 (Living Design Doc)

> **목적**: 앞으로 들어갈 기능과 수익 모델을 "구현 전에" 제대로 잡아두는 살아있는 설계 문서.
> 어떤 순서로 구현하든 여기서 정한 받침대(알림·장르태그·엔타이틀먼트·정산)를 먼저 깔면 중복 작업이 줄어든다.
> 결정이 서면 이 문서를 갱신하고, 구체 구현 스펙은 [`ToDoList.md`](../ToDoList.md)·`docs/features.md`로 내려보낸다.
> 비전 기준: 작가가 만화를 올리고 독자가 읽되, **작가↔독자 소통·커뮤니티·개인화**까지 갖춘 네이버웹툰류 플랫폼.

---

## 1. 현재 토대 (이미 구현 — 재활용 대상)

작품(Series)·회차(Episode, 이미지·조회수·예약발행)·**좋아요**·**회차 댓글**·**구독**·**읽음/이어보기(ReadLog)**·**북마크**·검색/정렬·문의(Inquiry)·역할(독자/작가/관리자)·JWT 인증·이미지 스토리지(`ObjectStorage` 추상화, 로컬/S3)·테마 콘솔.
→ 상세는 `docs/features.md`. **신규 기능은 이 위에 얹는다.**

핵심 패턴(따를 것): enum `@Enumerated(STRING)`, LAZY 연관 + 서비스 `@Transactional` 내 DTO 평탄화(OSIV off), Flyway 스키마, Entity 대신 DTO 노출, 권한검증은 서비스, `@PreAuthorize`로 ADMIN 게이팅, 이미지 키 UUID, `ObjectStorage`처럼 외부의존은 인터페이스로 추상화.

## 2. 기능 로드맵

각 항목: **목적 / 핵심 기능 / 기존 재활용 / 의존성 / 열린 결정**.

### 2.1 받침대 (Foundations) — 먼저 깔수록 이득

#### 🔔 알림 센터 (Notification) — **최우선 받침대**
- **목적**: 신규회차·작가소식·내 글/댓글 반응·문의답변·베스트등극을 한 곳에서. 팔로우·커뮤니티·문의가 전부 여기에 의존.
- **핵심**: `Notification(user, type, 대상ref, read, createdAt)` + fan-out 생성 + `GET /api/me/notifications`(페이징) + 읽음 `PATCH .../{id}/read` + 미읽음 개수(뱃지).
- **재활용**: 신규회차 fan-out은 구독자(`SubscriptionRepository`), 발행 훅(`EpisodeService.upload`/`EpisodePublisher.publishDueEpisodes`).
- **의존성**: 없음(독립). **이게 먼저면 팔로우 소식·커뮤니티 반응 알림이 공짜로 얹힘.**
- **열린 결정**: 전달 방식(폴링/SSE/WebSocket/외부푸시), fan-out 동기 vs 비동기(`@TransactionalEventListener`).
- **📄 상세 설계(데이터 모델·이벤트·API)**: **[`docs/design-notification.md`](design-notification.md)** — V14 스키마·엔티티·fan-out·멱등까지 확정. 전달방식 표는 [`ToDoList.md`](../ToDoList.md).

#### 🏷️ 장르·태그 (Genre/Tag)
- **목적**: 지금은 연령등급·상태뿐. 탐색·랭킹·관심설정·추천의 공통 받침대.
- **핵심**: 작품에 장르(enum: 로맨스/액션/판타지/일상/스릴러…) + 자유 태그(다대다). 검색·필터에 추가.
- **재활용**: 기존 검색(`search`)·정렬에 태그/장르 필터 추가.
- **의존성**: 없음. 독자 서재·추천·랭킹이 이걸 전제로 함.
- **열린 결정**: 장르=고정 enum vs 관리자 관리형 마스터. 태그=자유입력 vs 승인형.

#### 🚨 신고·제재·안전 (Moderation)
- **목적**: 커뮤니티가 생기는 순간 필수. 글/댓글/유저/작품 신고, 차단, 운영자 제재, 금칙어.
- **핵심**: `Report(reporter, 대상type+id, 사유, status)` + 관리자 처리 큐(콘솔 "신고 관리") + 차단(`Block`) + 금칙어 필터.
- **재활용**: 관리자 콘솔 패턴(문의 관리와 동일 구조), `@PreAuthorize(ADMIN)`.
- **의존성**: **커뮤니티와 동반 출시 필수.**
- **열린 결정**: 자동 숨김 임계치(신고 N건 자동 블라인드) 도입 여부.

#### 👤 프로필·계정 (Profile/Account)
- **목적**: 지금 설정 모달에 **자리만 있고 비어 있음**(닉네임/비번 변경 API 없음). 소셜의 기반(아바타·소개).
- **핵심**: 프로필(닉네임·아바타·소개·공개범위), 닉네임/비밀번호 변경 API, **작가 전환 신청 플로우**(현재 관리자 수동 부여 → 셀프 신청+심사).
- **재활용**: 설정 모달(이미 있음)에 섹션 추가, 이미지 스토리지(아바타).
- **의존성**: 팔로우/커뮤니티/멤버십이 프로필을 전제로 함.

#### 📊 작가 통계 (Creator Analytics)
- **목적**: 작가 리텐션의 핵심. 조회·좋아요·구독·댓글·완독률 추이, 회차별 성과.
- **핵심**: 집계 대시보드(콘솔 작가 영역). 광고 로열티 정산의 지표 원천도 됨.
- **재활용**: 기존 viewCount·like·subscription 카운트.
- **의존성**: 광고 로열티(4.수익)와 지표를 공유.

### 2.2 소셜 — 팔로우 + 작가 소식 피드
- **목적**: 독자↔작가 소통의 시작점.
- **핵심**: `Follow(reader→author)` + 작가 프로필 **소식 피드**(공지/근황 글) → 팔로워에게 알림. 소식에 댓글 가능.
- **재활용**: 알림 센터(소식 fan-out), 댓글 패턴.
- **의존성**: 알림 센터 + 프로필. **멤버십(유료 팔로우)의 토대** → 4.수익과 직결.
- **열린 결정**: 소식 피드 = 작가 전용 글 vs 작품별 공지 분리.

### 2.3 커뮤니티 — 게시판 + 대댓글 + 베스트
- **목적**: 독자/작가가 자유롭게 소통·추천하는 공간.
- **핵심**: 글(말머리: 추천/자유/팬아트/질문, 이미지 첨부) + **추천 → 베스트(HOT)**(추천수+시간 가중) + 인기/최신 정렬 + **댓글 + 대댓글** + 댓글 추천/베스트댓글 + 신고.
- **재활용**: 회차 댓글 구조 확장(현재 단방향 댓글 → parent 추가로 대댓글).
- **의존성**: **신고·안전 동반 필수**, 알림(내 글/댓글 반응).
- **열린 결정**:
  - 범위: **전역 커뮤니티 vs 작품별 게시판 vs 둘 다**.
  - 대댓글 깊이: **1-depth(네이버 방식) 권장** vs 무한 중첩.
  - 베스트 승격 공식·기준(추천수/시간/조회).

### 2.4 독자 개인화 — 서재 + 추천
- **목적**: 본 것·관심을 모아 보고, 취향 기반 추천.
- **핵심**: **내 서재**(구독·북마크·최근 본 작품 통합) + 관심 설정(선호 장르·태그·작가) + **개인화 추천 피드**.
- **재활용**: ReadLog(최근 본)·Bookmark(관심)·구독 이미 있음 → 화면화 + 장르·태그로 추천.
- **의존성**: 장르·태그.
- **열린 결정**: 추천 = **규칙기반(장르/태그 매칭) 단순 권장** vs 협업필터링(범위 큼).

### 2.5 탐색 — 랭킹·평점·캘린더
- **핵심**: 인기/급상승/요일별/장르별 랭킹, 작품 **별점(좋아요와 별개)**, 연재 캘린더(요일 — publishDays 있음), 신작/완결.
- **재활용**: viewCount·like·subscription·publishDays.
- **의존성**: 장르·태그(장르별 랭킹).

## 3. 수익 모델 설계

> 사용자 방향: **작가가 작품 단위로 수익 방식을 직접 설정**. 후원(일회/정기)·기다리면무료·유료 미리보기·전체무료·광고 로열티를 **복합** 적용. 지금은 사업화 아님 — **모델/뼈대만 설계, 결제는 mock부터**.

### 3.1 척추 — 작품별 "공개·수익 정책"
작가가 작품마다 정책 세트를 설정 → 독자 경험·수익이 갈림.
```
작품.공개정책 = {
  모드: 전체무료 | 기다리면무료 | 유료미리보기
  미리보기_잠금_화수: N          // 최신 N화 잠금
  무료전환_주기: 7일(작가 설정)   // 기다리면무료
  미리보기_가격: 코인 X           // 유료미리보기
}
```
→ 회차에 **FREE / LOCKED / PAID** 개념 추가. 현재 `ageRating·visible` 가드 **바로 그 자리**에 **접근권(entitlement) 가드**를 끼운다.

### 3.2 메커니즘 4종 (복합)
| 방식 | 동작 | 독자 결제 |
|---|---|---|
| **기다리면무료** | 스케줄러가 주기마다 최신 잠금화 1화씩 자동 무료 전환(EpisodePublisher 패턴 재활용) | ❌ (시간 잠금) |
| **유료 미리보기** | 잠긴 최신화를 코인으로 미리 열람 → 엔타이틀먼트 적립 | ✅ 코인 |
| **후원(일회/정기)** | 작품·작가 단발 후원 / **정기 후원 = 멤버십**(월정액) → 멤버 혜택(미리보기 무료·전용 소식·뱃지) | ✅ |
| **광고 로열티 풀** | 플랫폼 광고수익을 **참여지표(조회·구독·팔로우·추천·완독률)에 비례 차등 분배** | ❌(독자) / 정산 |

**시너지**: 정기 후원 = 멤버십 = **팔로우의 유료 상위 단계**. `팔로우(무료) → 멤버십(유료)` 사다리 → 2.2 소셜이 그대로 수익 인프라가 됨.

### 3.3 받침대 인프라 — **여기가 진짜 일**
- **코인/지갑(Wallet)**: 유료 미리보기는 코인(충전→소비). 후원은 직접결제 권장(분리).
- **엔타이틀먼트(접근권) 테이블**: 누가 어떤 회차 unlock / 멤버십 보유 → 회차 접근 가드 근거.
- **정산 원장(Ledger)**: 작가 수익(판매·후원·로열티) 적립 + 출금. **멱등·이중기록**(회계 정확성)이 생명.
- **결제 게이트웨이(PG)**: 실결제는 외부(토스/아임포트). **학습 단계엔 mock/내부코인으로 추상화**(`ObjectStorage` 추상화와 동일 전략).
- **유료 이미지 보호**: 유료 회차면 `/files` 정적서빙을 **인증 서빙으로 승격**(현재 URL 유추 가능).

### 3.5 ✅ 수익화 0단계 구현 완료 (STEP 28) — 결제 0, 작가 공개정책 + 기다리면무료
- **모델**: `Series.releasePolicy`(FREE_ALL/WAIT_FREE) + `waitFreeDays`(Integer nullable, 불변식 WAIT_FREE⟺>0). V22 backfill=FREE_ALL. 작가 `PATCH /api/series/{id}/release-policy`(CREATOR+소유). EpisodeStatus는 불변(잠금은 상태 아님).
- **잠금 = compute-on-read**(스케줄러·상태저장 0): `EpisodeAccessEvaluator.evaluate(series,episode,viewerId,isPrivileged,now)→AccessResult(accessible,lockReason{NONE,WAIT},freeAt)`. WAIT_FREE면 `freeAt=publishAt+waitFreeDays일`, `now>=freeAt`면 무료. 작가·admin 프리뷰는 항상 통과(publishAt-null 방어).
- **가드 배선**: getDetail(status 가드 직후)·getEpisodes(목록 락 주입)·**markRead(잠긴 회차 읽음 no-op — UP/이어보기/서재 신호 오염 방지)**. 잠긴 회차 = **200+locked**(이미지·조회수 비노출, ErrorResponse가 freeAt 못 실어 throw 대신 플래그). 정책전환은 소급 안 잠김(과거 회차 freeAt 이미 지남=직관적, 의도된 설계).
- **정렬·소급**: 익명열람 불가(인증 필수)라 viewerId non-null.

### 3.6 🔌 미래 확장점 (재작업 0의 seam — 이 자리에 끼운다)
| 미래 기능 | 끼우는 정확한 자리 | 0단계가 남긴 seam | 데이터 증분 |
|---|---|---|---|
| 엔타이틀먼트 테이블 | `EpisodeAccessEvaluator` **미래주입점 ①**(waitLocked 직전): `if(entitlementRepo.has(viewerId, episode.id)) return open()` | viewerId 이미 파라미터 | 테이블+리포 |
| 유료미리보기(PAID) | 주입점 ① 분기 + `LockReason.PAID` enum 상수 + `AccessResult.price` 필드 | enum/record 증분 자리 | `series.preview_price` V23 add column nullable |
| 코인/지갑 | getDetail의 PAID 분기 시 **별도 결제 유스케이스**(가드는 차감 모름=관심사 분리) | evaluate는 "열람 가능?"만 | Wallet 엔티티 |
| 정산원장(멱등·이중기록) | 결제 유스케이스 내부 tx, 가드 무관 | — | Ledger 테이블 |
| 멤버십 | 주입점 ①: `if(membership.covers(viewerId, series)) return open()` | series 이미 파라미터 | 멤버십 테이블 |
| PG(토스) | `PaymentGateway` 인터페이스(`ImageStorageService` Local/S3 분기 패턴, mock부터) | 설계만(코드 0) | — |
| 유료이미지 인증서빙 | `imageStorageService.urlFor` 승격 | 0단계는 잠기면 이미지 미반환이라 불필요 | — |

**정직한 표현**: 호출부 시그니처·반환구조·컨트롤러 골격은 **0재작업**. 단 PAID 가격 노출은 `AccessResult`+DTO에 `price` 필드 **추가**(스키마 증분) — "구조 0재작업 + 데이터 증분". `preview_price`는 지금 안 깐다(YAGNI — 사용처 없는 컬럼=검증·문서·테스트 부채).
**보류(0단계 밖)**: like/bookmark 잠금가드(저영향·제품결정), DB CHECK 제약(엔티티가 단일 쓰기경로라 충분), 무료전환 알림(필요 시 EpisodePublisher 스케줄러 병행).

### 3.4 수익 모델 열린 결정
- 설정 단위: **작품 단위 기본** + (옵션) 회차 오버라이드.
- 코인 vs 직접결제: **유료회차=코인 / 후원=직접결제** 분리 권장.
- 광고 로열티 **공식**: 단순 조회수면 어뷰징(조회 부풀리기) 위험 → **완독률·구독 등 질 지표 혼합 + 어뷰징 방지**.
- 플랫폼 **수수료율(take rate)**, 정산 주기·최소출금.

### 3.5 수익화 단계 (결제 0부터)
1. **0단계(결제 없음)**: 공개정책 = *전체무료 / 기다리면무료*. **결제 인프라 없이** 작가 설정 + 시간 잠금 unlock으로 수익화 뼈대(정책·회차잠금·스케줄unlock) 검증.
2. **1단계**: 코인 + 유료 미리보기(충전 mock → 엔타이틀먼트 + 원장).
3. **2단계**: 후원(일회 → 정기 멤버십) + 멤버 혜택.
4. **3단계**: 광고 로열티 풀 정산(작가 통계 지표 집계 + 배분).

## 4. 추천 빌드 순서 (의존성 기준)

```
[받침대]  🔔 알림센터 ── 🏷️ 장르·태그 ── 👤 프로필·계정
              │              │                │
              ▼              ▼                ▼
[소셜]    팔로우+작가소식 ──────────────┐
              │                          │
[커뮤니티] 게시판+대댓글+베스트 ── 🚨 신고·안전(동반)
              │
[개인화]  독자 서재 + 추천(장르·태그 기반)
              │
[탐색]    랭킹·평점·캘린더 ── 📊 작가 통계
              │
[수익]    0단계(작가설정 공개정책+기다리면무료, 결제0)
            → 1단계 코인·유료미리보기 → 2단계 후원·멤버십 → 3단계 광고로열티
```
**원칙**: 알림 센터 + 장르·태그를 먼저 깔면 소셜·커뮤니티·개인화·수익 지표가 전부 그 위에 얹혀 중복이 사라진다. 수익화는 **결제 없는 0단계(기다리면무료)**부터.
